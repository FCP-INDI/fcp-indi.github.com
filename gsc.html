GSoC @ CMI

Google Summer of Code with Child Mind Institute

Overview

The goal of the Configurable Pipeline for the Analysis of Connectomes (CPAC) project is to make cutting edge analyses of very large neuroimaging datasets accessible to researchers from a variety of backgrounds. This is accomplished through providing simplified user interfaces that permit processing pipelines to be easily constructed and executed on high performance computing infrastructures. At its core, CPAC uses Nipype to link together functions from popular open source neuroimaging analysis software packages (e.g. AFNI, FSL, ANTS, etc...) into processing pipelines that are then executed in parallel on high performance computing architectures such as clusters and multicore systems. CPAC currently implements state-of-the-art preprocessing and analysis strategies for functional magnetic resonance imaging data (fMRI), but our goal is to extend it to other neurimaging modalities (e.g. structural, diffusion, perfusion MRI) in the near future.

There are a variety of different ways to contribute to CPAC project including improving the application infrastructure, adding new data analysis methods, adding support for new neuroimaging technques, and developing better visualizations for the data. We list a few ideas for each of these areas below, but are also open to blue sky projects that are outside of the areas we have specified. We encourage involvement from individuals with a diversity of backgrounds and skill levels.

Proposal Format

Proposals should be formatted using Arial or Times New Roman font that is no smaller than 11pt. Page margins should be 0.5 inches on each side. Please include the following components in the application, and please respect the page limits.

  1. Brief Student Statement (1-2 pages)

    • Who are you? What are you studying?
    • What experience do you have with the tools you will be using?
    • What are your plans/hopes for the future?
    • What are your plans for the summer? How many hours a week will you commit to this project? Do you have any other commitments or plans during the summer (i.e. summer job, vacation plans)?
    • Where do you live? If near New York City, would you be willing to work on site at CMI headquarters (445 Park Ave), or periodically come in for meetings?
    • If your native language is not English, are you comfortable working closely with a supervisor who speaks English?

  2. Current CV or resume

  3. Project Propsal (2 pages)

    • Provide a high-level description of the proposed project
    • What do you plan to accomplish? What will not be performed?
    • What tools do you plan to use to accomplish the project?
    • What are appropriate milestones for the project? What is the project timeline?
    • How do you plan to test the project? This should be a real-world test to prove that the developed project works as expected.

Idea List

Basic Application Development (Infrastructure)

Real-time dashboard for currently executing analyses

  • Brief explanation: CPAC can run an extensive array of neuroimaging pipelines, which contain several steps. At present, a log file is kept of the steps executed as well as any standard error and output, which can be difficult to use in keeping track of all the steps being executed. The goal of this project would be to create a more elegant status page that would allow users to better monitor the execution of analyses.
  • Expected results: An html page that indicates information about each step including the status (e.g., running/error/done), execution time, possible output, etc. This page would potentially use javascript to update the status of the analyses in real-time.
  • Knowledge Prerequisite: Python, html, css, and javascript
  • Skill level: novice or medium
  • Mentor: Steve Giavasis and Daniel Clark

Read in data from database using RESTful API

  • Brief explanation: CPAC currently assumes that all of the input data is available on the local filesystem, and determines data locations using globbing. This project would enhance CPAC capabilities by allowing it to retrieve data locations and the data itself from neuroinformatics infrastructures using a RESTful interface. The RESTful API will be provided by us.
  • Expected results: Ability to read in data locations and data from a RESTful API.
  • Knowledge Prerequisite: Python, RESTful APIs
  • Skill level: novice, medium
  • Mentor: Steven Giavasis, Daniel Clark

Revamp and Revitalize our GUI for creating analysis pipelines

  • Brief explanation: CPAC was initially created as a command-line interface with a GUI that wraps around this interface. We have begun to encounter issues with our current GUI backend (wxPython) and have found that it is not very extensible to our needs, has lags in load time, and is difficult to install on certain systems. Here, we would be looking to move our current GUI interface to another system.
  • Expected results: Create a new GUI interface to replace the current wxPython-based one as well as introduce new features to the GUI that would make it easier to interface with the command-line interface.
  • Knowledge Prerequisite: Familiarity with gui-design
  • Skill level: novice to high
  • Mentor: Steve Giavasis, Daniel Clark

Make basic installation packages using Docker, PIP, or easy_install

  • Brief explanation: Like many open-source tools, CPAC has several dependences. Currently, a more manual install process is required to install CPAC and all its prerequisite tools. The goal would be to put together a package installer for CPAC that makes it easier for users to get up and running.
  • Expected results: A simple installer using Docker, PIP, or easy_install that could be run by a user with minimal hassle to install CPAC and its dependencies.
  • Knowledge Prerequisite: Python
  • Skill level: novice
  • Mentor: Steve Giavasis, Daniel Clark

Data Analysis

Supervised machine learning techniques to characterize clinical data.

  • Brief explanation: For each participant in a neuroimaging study we typically have several labels corresponding to things like whether or not they suffer from a disorder, sex, age, and others. The goal of this project is to implement machine learning algorithms to learn classifiers capable of prediciting the label from derivatives calculated from the neuroimaging data. For example, we might use grey matter density measures extracted from structural scans to predict age or sex.
  • Expected results: Supervised machine learning workflow that can work with any statistical derivative that CPAC generates.
  • Knowledge Prerequisite: Python, machine learning libraries such as scikit.learn and machine learning algorithms.
  • Skill level: medium to high
  • Mentor: Cameron Craddock, Steven Giavasis, Daniel Clark

Add any new analysis to CPAC - for more advanced / neuroimaging initiates

  • Brief explanation: CPAC includes many different analyses that are aimed at identifying relationships between brain function and structure and different individual qualities or traits. For example we may be interested in determining how brain activity in a particular brain region correlates with an individuals ability to read. CPAC has several analysis already implemented, but we would like to have more. This would be a great opportunity for a data-analysis expert to apply their favorite algorithm to neuroimaging data.
  • Expected results: Group level analysis that can be performed on any statistical derivative that CPAC genearates.
  • Knowledge Prerequisite: Python, statistical libraries such as Pandas, data analysis
  • Skill level: high
  • Mentor: Cameron Craddock, Steven Giavasis, Daniel Clark

Performance optimization of existing algorithms used to analyze neuroimaging data

  • Brief explanation: Performing data analyses on data from a thousand or more observations can take a long time. Particularly when each one of those observations contains tens of thousands of features. The goal of this project is to refactor current CPAC analaysis pipelines to make them much more efficient.
  • Expected results: Optimized code that can obtain same results as original results, in reduced time or with less memory.
  • Knowledge Prerequisite: Python optimization, numerical optimization
  • Skill level: medium, high
  • Mentor: Steven Giavasis, Daniel Clark

Visualization

Data visualization tools for the connectome

  • Brief explanation: The brain's connectome is complex and multidimensional. We need to apply the lastest in data visualization tools, like circos, d3 and 3D-embedded pdfs to graphically communicate our results. Help us modernize the standards in scientific publishing and data interaction software. This project provides exposure to the challenges in neuroimaging research, and a way to explore visualization tools that are available as free & open software.
  • Expected results: A software package which automates visual feedback for scientists and integrates with existing communcation media.
  • Knowledge Prerequisite: Basic Python, or javascript
  • Skill level: novice
  • Mentor: Aimi Watanabe, Steve Giavasis, Daniel Clark

WebApp for exploring brain function and anatomy

  • Brief explanation: Neurosynth is a product of neuroimaging scientists interested in the power of data exploration. As a open source alternative of The BrainMap Project, neurosynth compiles previous publications in cognitive neuroscience to extract themes and topics through natural language processing. Data from these papers are also collected, thereby linking brain-behavior relationships within the overarching database. Likewise, you can create your own webapp to help scientists and interested citizens understand the interaction of mind and body.
  • Expected results: A robust web-application for data exploration.
  • Knowledge Prerequisite: Basic javascript, html, & css
  • Skill level: novice to medium
  • Mentor: Aimi Watanabe, Steve Giavasis, Daniel Clark

Neuroscience

Add processing and analysis workflows for an usupported imaging modality

  • Brief explanation: CPAC currently supports MRI and resting state fMRI data, but we would like for it to support much more. Example modalities that are on our todo lists are diffusion tensor imaging (DTI), perfusion weighted imaging (ASL), myelin imaging, susceptibility weighted imaging and more. For most of these modalities this project would involve implementing preprocessing, calculating statistical derivatives from the data (i.e. FA, ADC for DTI, or mean CBF for perfusion), and making sure that the derivatives can work with the existing analysis algorithms.
  • Expected results: Workflows to perform preprocessing, calculate derivatiaves, and perform group level analyses.
  • Knowledge Prerequisite: Python, neuroimaging software tools (e.g. dipy for DTI), and basic knowledge of imaging modality to be addressed.
  • Skill level: medium to high
  • Mentor: Cameron Craddock, Steve Giavasis, Daniel Clark

System to receive and processing neuroimaging data in real-time

  • Brief explanation: Several emergent neuroimging experimental techniques rely on the ability to analyze the neuroimaging data as it is being acquired. For example, in real-time fMRI, brain activity is decoded from fMRI data as it is being collected and can be used to control a computer interface. We currently have software on the MRI scanner that will export fMRI data over a TCP/IP connection. This project would involve developing the software that receives the data, and uses other existing neuroimaging analysis software packages to analyze the data in real-time.
  • Expected results: Software that can receive and analyze neuroimaging data in real-time.
  • Knowledge Prerequisite: Python, network programming
  • Skill level: can accomodate students from novice to high
  • Mentor: Cameron Craddock, Steven Giavasis, Dan Clark
