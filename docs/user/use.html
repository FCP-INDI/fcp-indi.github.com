Using C-PAC¶

Ready Your Data¶

Before running C-PAC, you must generate a list of subjects and data to process.This is done by defining file structure templates in data_config.py and running the extract_data script. The latest version of data_config.py can be downloaded here.

It should be noted that C-PAC currently requires subject directories to be within a site level directory, even if your project does not contain data from multiple (data acquisition) sites. Users encountering errors when attempting to run extract_data.py may want to re-organize their data into a structure similar to that in the example presented below.

Define File Structure Templates¶

File structure templates are simply the full path to your data files, with the names of the site and subject directories replaced by wildcard placeholders (%s). For example, if the full paths to the data for subject_01 are:

/home/data/site_1/subject_1/anat/mprage.nii.gz
/home/data/site_1/subject_1/rest/rest.nii.gz

Then the file structure templates would would be:

/home/data/%s/%s/anat/mprage.nii.gz
/home/data/%s/%s/rest/rest.nii.gz

In cases where additional file and folder names contain subject specific information, asterisks can be used. For example, ../sub01_anat/sub01_mprage.nii.gz would become ../*_anat/*_mprage.nii.gz.

Configure Slice Timing Correction¶

If you wish to run Slice Timing Correction, you must specify a file that contains information about the slice acquisition order used for each subject.

If all subjects within a site have the same acquisition order:
Use the template scan_parameters.csv file available for download here.

If subjects within a site have differeing acquisition orders:
Use the template scan_parameters_multiscan.csv file available for download here.

Generate Subject Lists¶

When you are done setting up data_config.py, save your changes. In the terminal, navigate to the directory where you would like to store subject lists for use by C-PAC. Open iPython by typing ipython. From the iPython prompt, run one of the following commands.

If you are not running Slice Timing Correction, or all subjects within a site have the same acquisition order:

import CPAC
CPAC.utils.extract_data.run ('/path/to/data_config.py')

If you are running Slice Timing Correction and subjects within a site have differing acquisition orders:

import CPAC
CPAC.utils.extract_data_multiscan_params.run ('/path/to/data_config.py')

This will ouput three files needed to run C-PAC:

  • CPAC_subject_list.py - Subject list used when running preprocessing and individual-level analyses. Contains subject IDs and paths to their associated data files.
  • template_phenotypic.csv - Used by the FSL Model Script.
  • subject_list_group_analysis.txt - Subject list used when running group-level analyses.

Configure C-PAC¶

The config.py file is where you will specify which processing and analyses to run, and with which settings. This file is located in the /configs directory of your installation, or can be downloaded here.

For detailed instructions on how to configure a particular C-PAC function, see the following pages:

  • Computer Settings
  • Directory Setup
  • Anatomical Preprocessing
  • Functional Preprocessing
    • Slice Timing Correction
    • Temporal Filtering
    • Nuisance Corrections
  • Seed-based Correlation
  • Time Series Extraction
  • Regional Homogeneity
  • Voxel-mirrored Homotopic Connectivity
  • ALFF and fALFF
  • Network Centrality
  • Bootstrap Analysis of Stable Clusters
  • Connectome-wide Association Studies
  • Group Statistics
  • Smoothing

NOTE: If this is your first time using C-PAC, we strongly recommend that you go through the entire config.py file at least once. The default settings will not work for everyone, and it’s a good idea to consider the available options before running.

Running C-PAC¶

In the terminal, navigate to the crash directory you specified in config.py. Because of a bug in Nipype, C-PAC will output crash logs from wherever it is run.

Open iPython by typing ipython. From the iPython prompt, run:

import CPAC
CPAC.pipeline.cpac_runner.run ('/path/to/config.py', '/path/to/CPAC_subjectlist.py')

Running BASC, CWAS, and Group Statistics¶

In order to run group statistics, you must first run C-PAC to generate the desired measures for which you wish to calculate group differences.

For BASC and CWAS, C-PAC must first be run to transform functional images to MNI space.

Open iPython by typing ipython. From the iPython prompt, run:

import CPAC
CPAC.pipeline.cpac_group_runner.run ('/path/to/config.py', '/path/to/desired/strategy')
