Outputs¶

Software Concepts¶

Workflows¶

A workflow accomplishes a particular processing task (e.g. functional preprocessing, scrubbing, nuisance correction). Each workflow can be turned on or off in config.yml. A complete list of workflows can be found in the developer documentation.

Pipelines¶

A pipeline is a combination of workflows.

Strategies¶

A strategy is a set of preprocessing options. Specifically, a strategy is defined by segmentation thresholds, nuisance corrections, and scrubbing settings. Strategies are named based on their settings (e.g. _compcor_ncomponents_5_linear1.motion1.compcor1.CSF_0.98_GM_0.7_SCRUB_0.2_WM_0.98). Names are constructed as follows:

  • Number of principle components calculated by CompCor (if enabled)
  • Nuisance corrections selected.
  • CSF, Gray Matter, and White Matter segmentation thresholds.
  • Scrubbing threshold (if enabled)

Derivatives¶

Derivatives are the processed individual-level outputs of C-PAC. These include the following connectivity measures:

  • Seed-based Correlation Analysis (SCA) - Analyze the connectivity between brain regions.
  • Dual Regression - Compare large-scale networks.
  • Amplitude of Low Frequency Fluctuations (ALFF) and fractional ALFF (fALFF) - Measure the power of slow fluctuations in brain activity.
  • Voxel-mirrored Homotopic Connectivity (VMHC) - Investigate connectivity between hemispheres.
  • Bootstrap Analysis of Stable Clusters (BASC) - Quantify the stable characteristics of networks.
  • Regional Homogeneity (ReHo) - Measure the similarity of activity patterns across neighboring voxels.
  • Network Centrality - Analyze the structure of functional networks.
  • Connectome-wide Association Studies (CWAS) - Explore the relationship between patterns of functional connectivity and phenotypes.

C-PAC also makes it easy to export timeseries information from your data. You can choose to extract timeseries from regions of interest, and individual voxels.

For group analysis, C-PAC utilizes the powerful statistical tools in FSL to compare findings across groups.

Derivative Outputs¶

Derivatives for each subject can be found in the ../scan_<name> directory of each subject. This folder also contains the outputs of the registration and segmentation workflows.

Within each derivative folder will be sub-folders for each set of frequency filters set in config.yml, as well as a folder containing smoothed outputs.

Derivatives are output in both native space (*_img.nii.gz and *_map.nii.gz files), with values transformed to z-scores (*_Z_*.nii.gz) and in standard space (*_to_standard.nii.gz).
